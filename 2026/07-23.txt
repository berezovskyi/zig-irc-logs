bblack!~bblack@wikimedia/bblack-WMF
nickbors: abstractly (ignoring implementation details like Io.Threaded vs Io.Evented) - things only happen asynchronously by launching asynchronous "Tasks", via io.async() or io.concurrent().  If you're not calling those things, then everything is serialized.

1784836201
bblack!~bblack@wikimedia/bblack-WMF
Zig's Io.Evented (which is not ready for real use, IMHO, it's lagging behind Io.Threaded for now) is the closest thing to get poll()-like behavior (in terms of runtime consequences and thread models, etc), but it does this through the same Task abstraction, so your code ends up looking more like Threaded-style code in C.

1784836328
bblack!~bblack@wikimedia/bblack-WMF
there's also the io.operate() interface, which provides something that looks (in terms of your application code structure) more like a polling loop than threading, but it's also relatively-immature (and probably will only ever support a limited range of basic operations that are read/write-like)
